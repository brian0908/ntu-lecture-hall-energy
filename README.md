# 臺大教學館節能規劃 — 冷氣用電分析與政策缺口

**課程：** 環境與能源的資料科學  
**角色扮演：** 能源數據顧問公司，向台大校方提案  
**核心論點：** 台大目前沒有冷氣*使用*規範（只有購買規範），我們量化若落實政府 26°C 標準的節電潛力，並提出具體的政策建議

---

## 背景與問題框架

透過與台大總務處的訪談，確認了以下現況：

- **台大有購買規範，但沒有使用規範**：目前規定的是冷氣噸數與空間面積的比例，但對「幾度開、幾點關」沒有正式規定
- **過去曾有非正式提醒**（如 10 點後開、設定 28°C），但現在基本上沒有
- **教學館空調由教務處課務組管理**，非總務處；各系館各自管理
- **普通高壓空調電表**：總務處確認其後端電盤不由總務處維護，可能含非空調用電，不能視為純冷氣直接量測

這個政策缺口（governance gap）是本專案的核心切入點：*沒有規範，就無從檢驗合規性，也無從估算超額耗電有多嚴重。*

## 分析規劃

### 核心問題

台大沒有冷氣使用規範，若落實政府 26°C 標準，四大講堂乃至全校每年可節省多少用電？

### 分析架構

```
資料前處理
    ↓
EDA + 普通電表推算現行設定溫度（拐點分析）
    ↓
採用永續辦公室分解結果（各館舍 人員空調使用用電）
    ↓
交叉驗證：普通電表 vs 永續辦公室估算值比對
    ↓
26°C 反事實情境 → 節電量、減碳量、節費估算
    ↓
政策建議
```

### 各階段說明

**Stage 1｜資料前處理** `01_data_loading_cleaning.ipynb` ✅
- 載入所有電表資料（HTML-XLS 與 binary XLS 兩種格式）
- 清除異常值（負功率、極端峰值、電表重置）
- 補插短暫缺值（≤3小時），長缺值保留為 NaN
- 加入時間特徵（季節、學期、週末）
- 匯出 `cleaned/*.parquet` 供後續使用

**Stage 2｜EDA + 設定溫度推算** `02_eda.ipynb`
- 各講堂年度、月度用電趨勢（2016–2025）
- 用電熱力圖（月份 × 時段）
- 學期 vs 寒暑假、平日 vs 假日負載曲線比較
- **核心分析：** 普通 AC kW vs 室外溫度散佈圖，找出拐點 → 推算現行有效設定溫度（作為反事實計算的關鍵輸入）

**Stage 3｜永續辦公室方法採用與驗證** `03_sustainability_method.ipynb`
- 載入 `館舍用電基礎值.xlsx`，提取四個目標館舍的 `人員空調使用用電`
- 還原永續辦公室的分解邏輯（Dayoff × 溫度條件 × 75th percentile），確認我們理解其方法
- **交叉驗證：** 比對普通教學館的估算值（117 kW）與普通高壓空調電表在相同條件（熱天下午上課日）的實測分佈；記錄吻合程度與不確定性來源

**Stage 4｜26°C 反事實計算** `04_counterfactual.ipynb`
- 利用 CDH 比例縮放估算節電比例：`savings_fraction ≈ 1 − CDH_26 / CDH_actual`
- 將節電比例套用到四個館舍的 `人員空調使用用電`，估算高峰小時節電量
- 年化：估算年度有效冷氣運行小時數（從學期表 + 天氣資料推算）
- 套用至 `館舍用電基礎值.xlsx` 全 145 棟，估算全校年度節電潛力
- 換算為電費（台電高壓電價）與碳排（能源局年度排放係數）
- 敏感度分析：設定溫度估算 ±1°C 的影響

**Stage 5｜結果與政策建議** `05_results.ipynb`
- 彙整三層次結果：4 個目標講堂 → 全校 145 棟
- 提出三個政策槓桿：
  1. 正式訂定 26°C 設定溫度標準（對標政府規定）
  2. 非上課時段強制關機規定（針對教務處課務組）
  3. 要求無冷氣電表的館舍加裝子電表（讓永續辦公室的估算有實測依據）

### 重要假設

| 假設 | 說明 |
|---|---|
| 現行設定溫度 | 從 普通 AC kW vs 室外溫度的拐點推算，非直接量測；不確定性 ±1–2°C |
| 永續辦公室估算值的定義 | `人員空調使用用電`（kW）為熱天上課下午的 75th percentile 需求值，非年度能源總量；年化需乘以有效運行小時數 |
| CDH 縮放線性假設 | 假設 AC 用電與冷卻度時數線性相關；進行 ±1°C 敏感度分析驗證穩健性 |
| 碳排放係數 | 使用能源局每年公布的電力排放係數（kg CO₂e/kWh），依年份對應 |
| 普通電表的純度 | 學校確認後端電盤非總務處管理，可能含非空調負載；交叉驗證步驟量化此不確定性 |

## 每次工作的流程

### 第一步：開啟專案

打開 VS Code → File → Open Folder → 選擇 `ntu-lecture-hall-energy` 資料夾

### 第二步：拉取最新版本（Pull）

點左側**原始檔控制**圖示（三個圓圈連線的圖示，或按 `Ctrl+Shift+G`）

點上方的 **⋯ → Pull**，取得最新版本

或在內建終端機（`` Ctrl+` ``）輸入：
```bash
git pull
```

### 第三步：開啟筆記本開始工作

在左側檔案列表點擊 `.ipynb` 檔案，VS Code 會直接開啟 Jupyter 筆記本介面，不需要另開瀏覽器。

### 第四步：完成後上傳（Push）

先清除筆記本輸出：點筆記本上方工具列的 **⋯ → Restart Kernel and Clear All Outputs → 存檔（Ctrl+S）**

然後點左側**原始檔控制**圖示：

1. 在「Changes」區塊，點 **＋** 號將檔案加入暫存（等同 `git add`）
2. 在上方輸入框填寫說明，例如：`新增溫度與用電散佈圖`
3. 點 **✓ Commit** 按鈕
4. 點 **Sync Changes**（或 **Push**）上傳

第一次 push 會跳出 GitHub 登入視窗，登入自己的帳號即可。
